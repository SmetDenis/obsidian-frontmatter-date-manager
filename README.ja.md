Obsidian でノートを編集すると、YAML frontmatter 内の `created`、`updated`、`viewed` の日付を自動で更新します。

## なぜこのプラグインが必要なのか？

- **手作業での日付メンテナンスは面倒です。** ノートを編集するたびに frontmatter の `created` と `updated` を手で更新するのは、ミスが起こりやすく、執筆の流れを妨げます。
- **Obsidian には frontmatter の日付管理機能が組み込まれていません。** Obsidian はファイルシステムレベルで `ctime`/`mtime` を追跡しますが、ノート内の日付プロパティを自動的に書き込んだり維持したりはしません。
- **同期ツールが誤った更新を引き起こします。** Obsidian Sync、iCloud、Syncthing、Dropbox、Git ベースの同期は、実際の内容変更がないままファイルを書き換えます。コンテンツハッシュがなければ、同期のたびに日付更新がトリガーされ、ノイズが発生し、最悪の場合は無限の同期ループにつながります。
- **テンプレートや自動化プラグインと競合します。** Templater、Daily Notes、QuickAdd などのプラグインは、ファイルを作成すると同時に変更します。設定可能な遅延がないと、テンプレートが完全に適用される前に日付が書き込まれ、誤った日付になってしまいます。
- **既存のボールトには日付がありません。** 数百から数千のノートを持つボールトでこのプラグインを導入する際は、1つずつノートを更新するのではなく、ファイルシステムの日付から一括で日付を埋める手段が必要です。
- **手入力ではフォーマットが不揃いになります。** ノートごとに `2024-01-15`、`Jan 15, 2024`、`15.01.2024` など、さまざまな表記が混在してしまいます。このプラグインは、ボールト全体で単一の設定可能なフォーマットを強制します。
- **「最後に開いた日時」を自動で追跡する手段がありません。** Obsidian はファイルがいつ変更されたかは追跡しますが、最後にいつ *読んだ* かという概念はなく、これを frontmatter に書き込む他のプラグインもありません。このプラグインは、ノートを開くたびに `viewed` の日付をオプションで記録でき、間隔反復学習・レビューのワークフロー・「何か月も見ていないものは？」といったダッシュボードのために、Dataview でクエリできるようになります。

## 機能

- ファイル変更時に `updated` フィールドを自動更新（`mtime` と同期）
- 新規ファイルに `created` フィールドを自動設定（`ctime` と同期）
- ファイルを開いたときに `viewed` フィールドを自動設定 - 他のプラグインにはない独自機能（デフォルトでは無効）
- 各ノートの編集頻度をカウント（`updated_count`、デフォルトでは無効） - Bases/Dataview で並べ替えやフィルタができ、最もよく編集したノートを見つけられるおおまかなアクティビティの指標
- カスタマイズ可能な日付フォーマット（[date-fns](https://date-fns.org/v4.1.0/docs/format) 構文を使用）
- IANA タイムゾーンのオートコンプリート付きタイムゾーンサポート
- 文字列および数値プロパティ型（数値は Unix タイムスタンプに便利）
- プレビューと検証付きの gitignore スタイルのファイルフィルタルール
- 更新間隔の最小値を設定可能
- 新規作成ファイルの遅延（Templater、Daily Notes などとの互換性）
- 実際の変更を検出する SHA-256 コンテンツハッシュ（同期ツールによる誤更新を防止）
- 変更検出モード: ノート本文のみ、プロパティのみ、または両方
- 変更検出からのプロパティ除外
- 日付更新後にコマンドを実行
- 各ファイル自身のディスク上の日付から一括で日付を埋める（ドライランプレビュー付き）
- すべてのノートでプロパティ名を変更（古い名前をプレビュー付きで移行）
- 既存の日付を別のフォーマットへ再フォーマット（古い形式を解析して新しい形式で書き込み、プレビュー付き）
- すべての一括プレビューはページネーション（前へ/次へ）され、影響を受けるすべてのファイルを表示（行数の上限なし）し、デスクトップでは差分全体を TSV ファイルとしてダウンロードできます（システムのダウンロード先に保存され、ボールト内には決して書き込まれません）
- コマンドパレットまたはステータスバーから自動更新を切り替え
- 自動再開付きで自動更新を5分間一時停止
- Obsidian のアプリ言語に従う多言語 UI - 英語とロシア語は手動で確認済み、加えて19言語のベースライン翻訳があり、すべてキー単位で英語にフォールバック
- デスクトップとモバイルで動作

## インストール

### コミュニティプラグイン

Obsidian で 設定 > コミュニティプラグイン > 閲覧 を開き、**Frontmatter Date Manager** を検索して、インストールをクリックします。

### 手動インストール

[最新リリース](https://github.com/SmetDenis/obsidian-frontmatter-date-manager/releases/latest)
から `main.js`、`manifest.json`、`styles.css` をダウンロードし、
`<vault>/.obsidian/plugins/frontmatter-date-manager/` に配置します。

## 使い方

プラグインはインストール後に自動で動作します。markdown ファイルを編集すると、現在の変更時刻で `updated` プロパティを更新します。`created` プロパティがない場合は、ファイルの作成時刻を設定します。任意で、各ノートを最後に開いた日時を記録するため、設定で `viewed` 日付を有効にできます。

動作は **設定 -> Frontmatter Date Manager** で設定します。

### コマンド

| コマンド                                | 説明                                             |
|----------------------------------------|---------------------------------------------------------|
| **現在のファイルのタイムスタンプを更新** | アクティブなノートのタイムスタンプ更新を手動でトリガーします |
| **自動更新のオン/オフを切り替え**          | タイムスタンプの自動更新を有効または無効にします           |
| **自動更新を5分間一時停止**    | 自動再開付きで更新を一時的に停止します         |

**ステータスバーインジケーター** - 現在の状態（`Paused` または `Paused (Xm)`）を表示します。クリックすると自動更新のオン/オフを切り替えます。

## 設定

| 設定                            | デフォルト                 | 説明                                                                      |
|------------------------------------|-------------------------|----------------------------------------------------------------------------------|
| 作成日を追跡                | `true`                  | まだ作成日を持たないノートに作成日を追加します                             |
| 作成日プロパティ                   | `created`               | 作成日を保存するプロパティ名                                   |
| 最終編集日を追跡             | `true`                  | ノートを編集するたびにこの日付を更新します                                      |
| 更新日プロパティ                   | `updated`               | 最終編集日を保存するプロパティ名                                |
| 編集回数をカウント                        | `false`                 | 編集のたびに1ずつ増える数値プロパティを追加します（正確な履歴ではなく、おおまかなアクティビティの回数） |
| 編集回数プロパティ                | `updated_count`         | 編集回数を保存するプロパティ名                                       |
| 最終閲覧日を追跡             | `false`                 | ノートを開くたびに日付を保存します                                        |
| 閲覧日プロパティ                    | `viewed`                | 最後に開いた日付を保存するプロパティ名                                |
| 日付フォーマット                        | `yyyy-MM-dd'T'HH:mm:ss` | 日付と時刻のフォーマット（[date-fns 構文](https://date-fns.org/v4.1.0/docs/format)）  |
| タイムゾーン                           | `""` (system)           | IANA タイムゾーン識別子。空の場合はシステムのタイムゾーンを使用                         |
| 数値のみの日付を引用符なしで保存 | `false`              | 数字のみのフォーマットで、引用符付きテキストの代わりに数値を出力します                     |
| 自動更新                        | `true`                  | ノートを編集すると日付を自動的に更新します                                  |
| 更新間隔の最小秒数    | `30`                    | 日付更新間の最小間隔                                            |
| スキップするファイルとフォルダ          | `""` (all files)        | gitignore スタイルのルール: 行は除外、`!` は再包含、`#` はコメント              |
| 変更検出（コンテンツハッシュ） | `true`                  | 内容が実際に変更されたときだけ日付を書き込みます（SHA-256 ハッシュ）             |
| 変更とみなす対象            | `body`                  | 更新をトリガーする対象: `body`、`frontmatter`、または `both`                          |
| これらのプロパティを無視            | `[]`                    | 変更検出で無視するプロパティ。カンマ区切りで一度に複数追加可能   |
| 新規ファイルの遅延                     | `5000` ms               | 新規作成されたノートを処理する前に待機します                                       |
| 起動時にキャッシュを自動構築     | `true`                  | プラグインの読み込み時に、キャッシュされていないノートの変更検出データを構築します             |
| キャッシュエントリの最大数              | `10000`                 | キャッシュがこの上限を超えると、最も古い未使用エントリが削除されます              |
| 更新後のコマンド               | `""` (none)             | 各日付更新後に実行する Obsidian コマンド                                   |

### 作成日より前の更新日

| 設定                          | デフォルト    | 説明                                                                                                                       |
|----------------------------------|------------|----------------------------------------------------------------------------------------------------------------------------------|
| `順序が逆の日付の修正方法`  | `disabled` | 最終編集日が作成日より前の場合の処理。自動編集に適用されます。`disabled` は検出のみを意味します。 |
| `わずかな差を無視（秒）` | `0`     | 差がこの値より小さいときは順序が逆の日付を無視します。サブ秒単位の時計のずれを抑えるのに便利です。                            |
| `順序が逆の日付を検索`        | _(アクション)_ | ノートをスキャンし（スキップルールに従います）、最終編集日が作成日より前のものを一覧表示します。修正はモーダル内で適用します。 |

利用可能な戦略: `作成日を最終編集日に設定`、`最終編集日を作成日に設定`、`両方を最新の日付に設定`。

## 日付フォーマットの例

| フォーマット文字列           | 出力例            |
|-------------------------|---------------------------|
| `yyyy-MM-dd'T'HH:mm:ss` | 2026-04-12T14:30:00       |
| `yyyy-MM-dd HH:mm:ss`   | 2026-04-12 14:30:00       |
| `dd.MM.yyyy HH:mm`      | 12.04.2026 14:30          |
| `t`                     | 1776268200 (Unix 秒) |
| `T`                     | 1776268200000 (Unix ミリ秒)   |

> **注意:** このプラグインは Moment.js ではなく **date-fns** を使用します。よくある移行: `YYYY` -> `yyyy`、`DD` -> `dd`。

## 最もよく編集したノートを見つける（任意）

**編集回数をカウント** をオンにすると、`updated_count` プロパティは好きなように並べ替えやフィルタができる単純な数値になります。プラグインは数値を書き込むだけで、ビューの作成はあなた次第です。例えば、最もよく編集するノートを表示する Dataview クエリ:

````markdown
```dataview
TABLE updated_count, updated
WHERE updated_count
SORT updated_count DESC
LIMIT 20
```
````

または Obsidian の **Bases** で、`updated_count` の降順で並べ替えます（任意で、最近の `updated` でフィルタします）。このカウントは正確な履歴ではなく、おおまかなアクティビティの指標で、機能を有効にした瞬間から始まります。キーストロークごとではなく、編集 *セッション* ごとに1回増えます。**更新間隔の最小秒数** のウィンドウ内での連続編集は1回として数えられます。**現在のファイルのタイムスタンプを更新** コマンドの実行もカウントされます。一括操作（埋め込み/再フォーマット/名前変更）はカウントを変えずに日付を書き換えます。

## FAQ

### 初回インストール

> 最初に有効にしたとき、プラグインは既存のすべてのノートを変更しますか？

いいえ。プラグインはファイルを編集したときだけそのファイルを処理します。初回読み込み時には、変更検出に備えて既存ファイルのハッシュキャッシュをバックグラウンドで構築しますが、この過程でタイムスタンプを書き込むことは決してありません。実際にノートを編集するまで、ボールトはそのまま保たれます。

> インストール前に書いたノートにタイムスタンプを追加するには？

設定 → 一括操作 → ファイル自身の日付から日付を設定 を使います。ディスク上の各ファイル自身の作成日と変更日を読み取り、ノートのプロパティに書き込みます。コミット前に確認できるよう、ドライランプレビューが付いています。デフォルトモードは「不足分のみ埋める」で、既存の日付は上書きされません。ボールトを iCloud や Obsidian Sync で同期している場合、ディスク上のこれらの日付が同期サービスによってリセットされている可能性があります - プレビューを注意深く確認してください。

> Templater / Daily Notes / QuickAdd を使っています。プラグインは競合しますか？

いいえ。プラグインは新規作成されたファイルを処理する前に5秒待機します（設定可能: 設定 → 動作 → 詳細設定 → 新規ファイルの遅延）。これにより、テンプレートプラグインが完了する時間が与えられます。

> 先にすべてのノートに手動でプロパティを追加する必要がありますか？

いいえ。ノートにまだプロパティがない場合、プラグインは次回編集時に `---` ブロックを作成し、日付を挿入します。プロパティがすでに存在する場合は、既存のものと並べて日付プロパティを追加します。

> Dataview に最適な日付フォーマットは？

デフォルトの `yyyy-MM-dd'T'HH:mm:ss`（ISO 8601）はそのまま動作します。Dataview はこれをネイティブに解析・並べ替え・比較できます。

> プラグインは Moment.js ではなく date-fns を使うとのことですが、これは影響しますか？

日付フォーマットをカスタマイズする場合のみ影響します。主な違い: 年には `yyyy`（`YYYY` ではなく）、日には `dd`（`DD` ではなく）を使います。Moment.js スタイルのフォーマットを検出すると、プラグインは設定画面にヒントを表示します。

### 日常的な使い方

> 「viewed」タイムスタンプを有効にしましたが、一部のノートに表示されません。

viewed タイムスタンプはファイルを開いたときだけ書き込まれます。機能を有効にしてから開いていないノートには、まだこのフィールドがありません。viewed の書き込みには、同じフィルタルールと最小間隔の設定が適用されます。

> タグやエイリアスを編集しましたが、`updated` が変わりませんでした。これはバグですか？

いいえ。デフォルトでは、変更検出はノート本文のみを見ます - プロパティブロックより下の変更だけが日付更新をトリガーします。プロパティの変更を含めるには、設定 → 変更検出 → 変更とみなす対象 を「両方」に切り替えます。

> 同期（iCloud / Obsidian Sync / Dropbox）は誤ったタイムスタンプを引き起こしますか？

いいえ。プラグインは SHA-256 ハッシュでファイルの内容を比較します。同期サービスが内容を変えずにファイルを書き換えた場合、ハッシュが一致するためタイムスタンプは更新されません。デフォルトで有効です。

> ノートの名前を変更したり移動したりしました。プラグインは追跡できなくなりますか？

いいえ。ハッシュキャッシュのエントリは自動的に新しいパスへ移行されます。既存のタイムスタンプは保持されます。

> 日付フォーマットを変更しました。古いタイムスタンプは変換されますか？

自動では変換されません。すべての値を標準化するには 設定 → 一括操作 → 日付を再フォーマット を使います。プラグインは既存のフォーマット（ISO 8601、ヨーロッパ式、米国式、数値タイムスタンプ）を自動検出し、現在のフォーマットで書き換えます。適用前にすべての変更をプレビューしてください。

> `01/05/2024` のような日付は1月5日か5月1日のどちらにも取れます。どうなりますか？

このようなあいまいな日/月の日付は、デフォルトでは変更されません - プラグインは決して推測しません。プレビューでいくつ見つかったかを表示し、ワンクリックの選択肢（日が先か、月が先か）を提供します。これはお使いのシステムの地域から事前提案されるので、何かが書き換えられる前にあなたが決めます。解釈が1つしかない日付（例: `25/12/2024`）は常に変換されます。

> プロパティの名前を変更しました（例: `created` → `date_created`）。既存のファイルはどうなりますか？

設定 → 一括操作 → プロパティ名を変更 を使います。古いプロパティ名と新しいプロパティ名を入力し、影響を受けるノートをプレビューしてから適用します。古いプロパティを削除するか、両方を残すかを選べます。

> タイムゾーンを変更しました。古いタイムスタンプは再計算されますか？

いいえ。同じ原則で、古い値はそのまま保たれます。新しい書き込みには新しいタイムゾーンが使われます。

> ノートの YAML frontmatter が壊れている場合はどうなりますか？

プラグインはそのファイルをスキップし、ファイルパスとエラーの詳細を含む通知を表示します。不正な YAML を持つファイルに書き込むことは決してありません。構文を修正すれば、次回の編集時にプラグインが処理します。

> 高速に保存しています。保存のたびにタイムスタンプが更新されますか？

いいえ。更新間には最小30秒の間隔（設定可能: 5-300秒）に加えて2秒のデバウンスがあるため、連続編集は単一のタイムスタンプ書き込みにまとめられます。

## 同期とバージョン管理

プラグインは、自身のデータディレクトリ（`.obsidian/plugins/frontmatter-date-manager/`）内にローカルキャッシュファイル `hash-cache.json` を保存します。このファイルには、コンテンツ変更検出に使う SHA-256 ハッシュが含まれます。起動時に自動的に再構築されるため、除外しても安全であり、推奨されます。

**除外する理由:** キャッシュはファイル編集のたびに更新されるため、複数のデバイスが個別にそれを変更し、頻繁な同期競合と不要なトラフィックを引き起こします。自動的に再構築されるので、同期しても利点はありません。

`.gitignore` に追加します:

```
.obsidian/plugins/frontmatter-date-manager/hash-cache.json
```

**Obsidian Sync** の場合: このファイルはすでに自動的に除外されています（Sync は `data.json` 以外のプラグインデータファイルを同期しません）。

**iCloud、Syncthing、Dropbox、その他のファイルベースの同期** の場合: プラグインディレクトリについて、同期ツールの無視/除外リストに `hash-cache.json` を追加してください。

## プライバシーと権限

このプラグインは完全にローカルです。バックエンドを持たず、ネットワークリクエストを行わず、いかなる種類のテレメトリや解析データも収集しません。コミュニティプラグインのスコアカードには、プラグインのコードが使用できる権限が一覧表示されます。このプラグインが各権限を何に使っているかを正確に説明します:

- **ボールト内の markdown ファイルを読み取ります。** 一括ツール（日付を埋める、プロパティ名を変更、日付を再フォーマット、順序が逆の日付を検索、変更検出キャッシュを再構築）はボールト全体にわたって動作するため、Obsidian の `getMarkdownFiles()` で markdown ノートを一覧表示します。プラグインは markdown 以外のファイルを列挙しないため（`getFiles()` は使用しません）、添付ファイル、画像、その他のバイナリには決して触れません。
- **設定された日付プロパティのみを書き込みます。** すべての変更は Obsidian の `processFrontMatter()` を通して行われます。これは設定した `created` / `updated` / `viewed` プロパティのみに触れ、ノート本文、キーの順序、コメント、無関係なプロパティには手を付けません。
- **自身のプラグインフォルダ内に1つのサイドカーファイルを書き込みます。** SHA-256 変更検出キャッシュ（`hash-cache.json`）は `.obsidian/plugins/frontmatter-date-manager/` 内に書き込まれ、ノート内には決して書き込まれません。
- **ローカルへのエクスポートのみ。** 「差分全体をダウンロード」ボタンは、ブラウザを通じて差分をローカルの `.tsv` ファイルとして保存し、ボールト内にはファイルを書き込みません。ファイルのダウンロードはデスクトップ専用です - モバイルでは画面上の表で差分全体を読めるようになっています。

## 言語

プラグインのインターフェースは Obsidian 自身の言語設定に自動的に従うため、別途設定する言語オプションはありません。21言語（アラビア語、ドイツ語、英語、スペイン語、ペルシア語、フランス語、インドネシア語、イタリア語、日本語、韓国語、オランダ語、ポーランド語、ポルトガル語、ブラジルポルトガル語、ロシア語、タイ語、トルコ語、ウクライナ語、ベトナム語、簡体字/繁体字中国語）の翻訳を同梱しており、翻訳がカバーしていないテキストは英語にフォールバックするため、UI が空白になることはありません。英語とロシア語は手動で確認済みで、それ以外はベースライン翻訳です。**改善を大歓迎します** - [`CONTRIBUTING.md`](CONTRIBUTING.md) の「Translations」をご覧ください。

## 開発

```bash
make              # Show all available commands
make install      # Install dependencies
make pre-commit   # Run all checks (format, lint, test, build)
make local-test   # Build and copy plugin to local vault
```

`make local-test` を使うには、シェル環境で `OBSIDIAN_VAULT_TEST` を設定するか、直接渡します: `make local-test OBSIDIAN_VAULT_TEST=/path/to/vault`。
